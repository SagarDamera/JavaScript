# Creational Patterns

- Factory Pattern
- Constructor Pattern
- Singleton Pattern
- Prototype Pattern

## 1.1 Factory Pattern

Factory Pattern is used to create objects based on a given type or condition.

Creating different notifications like email, SMS, or push notification.

```js
function notificationFactory(type) {
  if (type === "email") {
    return {
      send() {
        console.log("Sending email notification");
      },
    };
  }

  if (type === "sms") {
    return {
      send() {
        console.log("Sending SMS notification");
      },
    };
  }
}

const notification = notificationFactory("email");

notification.send();
```

### Is Factory Pattern used in React?

Yes. It can be used to return different components based on a condition.

```jsx
function AdminDashboard() {
  return <h2>Admin Dashboard</h2>;
}

function UserDashboard() {
  return <h2>User Dashboard</h2>;
}

function dashboardFactory(role) {
  if (role === "admin") {
    return <AdminDashboard />;
  }

  return <UserDashboard />;
}

function App() {
  return <div>{dashboardFactory("admin")}</div>;
}

export default App;
```

## 1.2 Constructor Pattern

Constructor Pattern is used to create multiple objects with the same structure.

Creating multiple user profiles with the same fields like name and role.

```js
function User(name, role) {
  this.name = name;
  this.role = role;
}

const user1 = new User("Sagar", "Developer");
const user2 = new User("John", "Manager");

console.log(user1);
console.log(user2);
```

### Is Constructor Pattern used in React?

Yes. It was used in React class components to initialize state.

```jsx
import React from "react";

class UserProfile extends React.Component {
  constructor() {
    super();

    this.state = {
      name: "Sagar",
    };
  }

  render() {
    return <h2>{this.state.name}</h2>;
  }
}

export default UserProfile;
```

## 1.3 Singleton Pattern

Singleton Pattern is used when we want only one instance of an object in the whole application.

Using one app configuration object across the application.

```js
const AppConfig = {
  appName: "My App",
  version: "1.0.0",
  apiUrl: "https://api.example.com",
};

Object.freeze(AppConfig);

console.log(AppConfig.appName);
console.log(AppConfig.apiUrl);
```

### Is Singleton Pattern used in React?

Yes. It can be used for shared configuration or common service objects.

```jsx
const ThemeConfig = {
  theme: "dark",
};

function Header() {
  return <h2>Current Theme: {ThemeConfig.theme}</h2>;
}

function App() {
  return <Header />;
}

export default App;
```

## 1.4 Prototype Pattern

Prototype Pattern is used to share methods between objects instead of creating separate copies for every object.

Multiple users can share the same login method.

```js
const userMethods = {
  login() {
    console.log(`${this.name} logged in`);
  },
};

const user1 = Object.create(userMethods);
user1.name = "Sagar";

const user2 = Object.create(userMethods);
user2.name = "David";

user1.login();
user2.login();
```

### Is Prototype Pattern used in React?

Not directly in modern functional React, but JavaScript classes and older React class components use prototype internally.

```jsx
import React from "react";

class Greeting extends React.Component {
  sayHello() {
    return "Hello Sagar";
  }

  render() {
    return <h2>{this.sayHello()}</h2>;
  }
}

export default Greeting;
```
