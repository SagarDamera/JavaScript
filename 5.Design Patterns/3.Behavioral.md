## 3.1 Observer Pattern

Observer Pattern is used when one object changes, and many other objects need to be notified automatically.

YouTube subscribers get notified when a channel uploads a new video.

```js
class Channel {
  constructor() {
    this.subscribers = [];
  }

  subscribe(user) {
    this.subscribers.push(user);
  }

  uploadVideo(videoName) {
    this.subscribers.forEach((user) => {
      user.notify(videoName);
    });
  }
}

class User {
  constructor(name) {
    this.name = name;
  }

  notify(videoName) {
    console.log(`${this.name} got notification for ${videoName}`);
  }
}

const channel = new Channel();

const user1 = new User("Meghana");
const user2 = new User("David");

channel.subscribe(user1);
channel.subscribe(user2);

channel.uploadVideo("JavaScript Design Patterns");
```

### Is Observer Pattern used in React?

Yes. React state updates are similar to Observer Pattern. When state changes, React automatically updates the UI.

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>Count: {count}</h2>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

## 3.2 Publisher-Subscriber Pattern

Publisher-Subscriber Pattern is used when one part of the application publishes an event, and other parts listen to that event.

A news app publishes breaking news, and many users receive the update.

```js
class EventBus {
  constructor() {
    this.events = {};
  }

  subscribe(eventName, callback) {
    if (!this.events[eventName]) {
      this.events[eventName] = [];
    }

    this.events[eventName].push(callback);
  }

  publish(eventName, data) {
    this.events[eventName].forEach((callback) => {
      callback(data);
    });
  }
}

const eventBus = new EventBus();

eventBus.subscribe("orderPlaced", (data) => {
  console.log("Email sent for order:", data.orderId);
});

eventBus.subscribe("orderPlaced", (data) => {
  console.log("Inventory updated for order:", data.orderId);
});

eventBus.publish("orderPlaced", { orderId: 101 });
```

### Is Publisher-Subscriber Pattern used in React?

Yes. It can be used for communication between unrelated components using an event bus.

```jsx
const eventBus = {
  events: {},

  subscribe(eventName, callback) {
    this.events[eventName] = this.events[eventName] || [];
    this.events[eventName].push(callback);
  },

  publish(eventName, data) {
    this.events[eventName]?.forEach((callback) => callback(data));
  },
};

function Button() {
  return (
    <button onClick={() => eventBus.publish("message", "Hello Meghana")}>
      Send Message
    </button>
  );
}

function Message() {
  eventBus.subscribe("message", (text) => {
    console.log(text);
  });

  return <h2>Check console</h2>;
}

function App() {
  return (
    <div>
      <Button />
      <Message />
    </div>
  );
}

export default App;
```

## 3.3 Strategy Pattern

Strategy Pattern is used when we have multiple ways to perform a task, and we choose one strategy based on condition.

A payment system can use different payment methods like credit card, PayPal, or UPI.

```js
const paymentStrategies = {
  creditCard(amount) {
    console.log(`Paid ${amount} using Credit Card`);
  },

  paypal(amount) {
    console.log(`Paid ${amount} using PayPal`);
  },

  upi(amount) {
    console.log(`Paid ${amount} using UPI`);
  },
};

function pay(method, amount) {
  paymentStrategies[method](amount);
}

pay("creditCard", 100);
pay("paypal", 200);
pay("upi", 300);
```

### Is Strategy Pattern used in React?

Yes. It can be used to render different UI or run different logic based on a selected option.

```jsx
const roleMessages = {
  admin: "Welcome Admin",
  user: "Welcome User",
  guest: "Welcome Guest",
};

function WelcomeMessage({ role }) {
  return <h2>{roleMessages[role]}</h2>;
}

function App() {
  return <WelcomeMessage role="admin" />;
}

export default App;
```

## 3.4 Command Pattern

Command Pattern is used to wrap an action inside an object or function, so we can execute it later.

A TV remote button stores a command like turn on, turn off, or increase volume.

```js
function turnOnLight() {
  console.log("Light turned on");
}

function turnOffLight() {
  console.log("Light turned off");
}

const commands = {
  turnOn: turnOnLight,
  turnOff: turnOffLight,
};

function executeCommand(commandName) {
  commands[commandName]();
}

executeCommand("turnOn");
executeCommand("turnOff");
```

### Is Command Pattern used in React?

Yes. Button click handlers are simple examples of Command Pattern because we pass actions and execute them later.

```jsx
function Toolbar() {
  const commands = {
    save() {
      console.log("Saved");
    },

    delete() {
      console.log("Deleted");
    },
  };

  return (
    <div>
      <button onClick={commands.save}>Save</button>
      <button onClick={commands.delete}>Delete</button>
    </div>
  );
}

export default Toolbar;
```
