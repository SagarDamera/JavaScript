# Topic: Virtual DOM

- Virtual DOM is a lightweight JavaScript representation of the real DOM.

- React updates the Virtual DOM first, then efficiently updates only changed parts in the real DOM.

- Re-render does not always mean real DOM update; React compares old and new Virtual DOM first.

- This comparison process is called reconciliation.

- Keys help reconciliation work correctly when rendering lists.

- Virtual DOM improves developer experience, but performance still depends on good component design.

- Common mistake: thinking Virtual DOM makes every React app automatically fast.

- Avoid unnecessary re-renders in heavy components using proper state placement and memoization when needed.

- Interview trap: Virtual DOM is not the browser DOM; it is React’s in-memory UI representation.

- Real-world relevance: helps React manage complex UI updates predictably and efficiently.

## One-Line Final Recall

The Virtual DOM is a conceptual name for React’s in-memory description of the UI. In modern React, JSX creates React elements, and React uses those elements to build/update Fiber trees. During rendering, React creates a work-in-progress Fiber tree, compares it with the current Fiber tree, and then commits the final changes to the real DOM.
