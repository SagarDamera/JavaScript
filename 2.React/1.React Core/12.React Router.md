# Topic: React Router v7

- React Router v7 handles routing, navigation, nested layouts, data loading, and form actions.

- Use `createBrowserRouter` and `RouterProvider` for data routers and modern route setup.

- Use nested routes with `Outlet` to share layouts like dashboard, navbar, or sidebar.

- Use `loader` to fetch route data before rendering the route component.

- Use `action` to handle form submissions and mutations like create, update, or delete.

- Use `Link` for navigation instead of normal `<a>` to avoid full page reloads.

- Use `useNavigate` for programmatic navigation after login, submit, or redirect logic.

- Common mistake: fetching all route data inside `useEffect` when `loader` is a better fit.

- Interview trap: route params, search params, loader data, and navigation state are different concepts.

- Real-world use: dashboards, auth routes, product pages, nested layouts, protected routes, and CRUD apps.
