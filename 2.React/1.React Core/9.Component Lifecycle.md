# Topic: Component Lifecycle

- Component lifecycle means the stages a component goes through: mount, update, and unmount.

- Mount happens when the component is first added to the screen.

- Update happens when props, state, or parent re-render causes the component to render again.

- Unmount happens when the component is removed from the UI.

- In functional components, `useEffect` handles lifecycle-related work.

- `useEffect(() => {}, [])` runs after the first render, similar to mount behavior.

- `useEffect(() => {}, [value])` runs when the dependency value changes.

- Return a cleanup function from `useEffect` to handle unmount cleanup.

- Common mistake: missing cleanup for timers, subscriptions, event listeners, or API cancellation.

- Interview trap: rendering and effects are different; effects run after React updates the DOM.

## One-Line Final Recall

React lifecycle is about how components mount, update, and unmount, usually handled with `useEffect` in modern React.
