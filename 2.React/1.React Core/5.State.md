# Topic: State

- State is component-owned data that changes over time and affects UI rendering.

- Updating state causes React to re-render the component.

- Never mutate state directly; always use the setter function from `useState`.

- State updates are asynchronous and may be batched by React.

- Use functional updates `const [count, setCount] = useState(prev => prev + 1)` when new state depends on previous state .

- Keep state minimal; do not store values that can be derived from props or existing state.

- Lifting state up helps share data between sibling components through a common parent.

- Common mistake: expecting state to update immediately after calling the setter.

- Interview trap: state is local to a component unless passed down as props or moved to shared/global state.

- Real-world use: state manages form values, toggles, loading status, selected items, and UI interaction.

## One-Line Final Recall

State is mutable component data that drives UI changes and triggers re-rendering when updated.
