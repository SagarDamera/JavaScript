# Topic: React Hook Form & Zod

- React Hook Form manages forms with less re-rendering compared to fully controlled form state.

- It works well for large forms because it uses refs and isolates field updates.

- Zod is used for schema-based validation and TypeScript-friendly type safety.

- Use `zodResolver` to connect Zod validation with React Hook Form.

- Keep validation rules in the Zod schema, not scattered across JSX.

- Use `register` for simple inputs and `Controller` for controlled components like custom selects or date pickers.

- Common mistake: forgetting to show validation errors from `formState.errors`.

- Common mistake: using `Controller` for every field when `register` is enough.

- Interview trap: React Hook Form does not update React state on every keystroke like normal controlled inputs.

- Real-world use: login forms, checkout forms, multi-step forms, admin forms, and complex validation flows.

## One-Line Final Recall

React Hook Form handles performant form state, while Zod defines clean, reusable, type-safe validation rules.
