# Topic: Props

- Props are read-only inputs passed from parent to child components.

- Use props to make components reusable and dynamic instead of hardcoding values.

- Never mutate props inside a child; update data in the parent and pass the new value down.

- Props flow one-way: parent → child. Child sends data back using callback props.

- Destructure props for cleaner code, but avoid over-destructuring if it reduces readability.

- Passing objects, arrays, or functions as props can cause unnecessary re-renders if references change.

- Use default values for optional props to avoid undefined errors.

- Common mistake: copying props into state without a real reason; it can create stale UI bugs.

- Interview trap: props are not state; changing props depends on parent re-rendering.

- Real-world use: props help build reusable components like Button, Card, Modal, and FormField.

## One-Line Final Recall

Props are read-only data passed from parent to child to make components reusable and dynamic.
