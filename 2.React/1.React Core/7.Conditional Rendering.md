# Topic: Conditional Rendering

- Conditional rendering means showing different UI based on state, props, or data.

- Common ways: `if`, ternary operator, `&&`, early return, or separate helper function.

- Use ternary when you need `if/else` UI inside JSX.

- Use `&&` only when rendering something or nothing.

- Be careful with `0 && <Component />`; React may render `0` on the screen.

- Use early return for loading, error, empty, or unauthorized states.

- Avoid deeply nested ternaries; they make JSX hard to read and maintain.

- Keep conditions simple; move complex logic outside JSX into variables.

- Common mistake: mixing too much business logic directly inside JSX.

- Real-world use: loading spinners, error messages, empty states, role-based UI, and feature flags.

## One-Line Final Recall

Conditional rendering lets React show different UI based on current data, state, or user actions.
