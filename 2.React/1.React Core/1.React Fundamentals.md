# Topic: React Fundamentals

- React is a component-based `UI library`, not a full framework.
- Components should be reusable, predictable, and focused on one responsibility.
- JSX is syntax sugar for React.createElement; it looks like HTML but is JavaScript.
- Props are used to pass data from parent to child and should be treated as read-only.
- State is used for data that changes over time and causes UI re-render.
- React updates the UI by re-rendering components when state or props change.
- Always use `setState` setter functions never mutate state directly.
- React uses the Virtual DOM and reconciliation to efficiently update only required DOM changes.
- `Keys` help React identify list items correctly during re-rendering.
- Keep rendering logic pure; avoid side effects directly inside the component body.

## One-Line Final Recall

React builds UI using reusable components where props pass data, state handles changes, and re-rendering keeps the UI in sync.
