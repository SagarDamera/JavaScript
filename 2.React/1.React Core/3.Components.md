# Topic: React Components

- Components are reusable UI building blocks; keep them focused on one clear responsibility.

- Prefer functional components in modern React; class components are mostly legacy but may appear in older codebases.

- Props are inputs to a component; never mutate props inside the child component.

- State belongs inside the component when data changes over time and affects UI rendering.

- Component names must start with a capital letter so React treats them as custom components.

- Keep components small and readable; split large UI into presentational and container/logical components when needed.

- Avoid putting too much business logic directly inside JSX; move logic to variables, helpers, or hooks.

- Re-render happens when props, state, or parent render changes; avoid unnecessary re-renders in heavy components.

- Common mistake: creating components inside another component, which can reset state and hurt performance.

- Interview trap: props flow one-way from parent to child; child communicates back using callback functions.

## One-Line Final Recall

React components are reusable UI units that receive props, manage state when needed, and re-render based on data changes.
