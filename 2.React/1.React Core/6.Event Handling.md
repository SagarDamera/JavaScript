# Topic: Event Handling

- React events use camelCase names like `onClick`, `onChange`, and `onSubmit`.

- Pass a function reference to an event handler, not the result of calling it.

- Use event handlers to update state, call APIs, validate input, or trigger UI changes.

- The event object gives access to details like `event.target.value`.

- Use `event.preventDefault()` to stop default browser behavior, especially in forms.

- For passing arguments, wrap the handler in an arrow function.

- Avoid writing heavy logic directly inside JSX; keep handlers readable.

- Common mistake: calling `onClick={handleClick()}` which runs during render.

- Interview trap: React event handlers run after render when the user interacts, not during JSX creation.

- Real-world use: handling form input, button clicks, dropdown changes, modal actions, and keyboard events.

## One-Line Final Recall

Event handling in React connects user actions to state updates, UI behavior, and business logic through handler functions.
