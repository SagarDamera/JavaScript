# Topic: JSX

- JSX is a JavaScript syntax extension used to describe UI in React.
- JSX looks like HTML, but it gets converted into React.createElement().
- A component must return one parent element; use fragments `<>...</>` when no wrapper is needed.
- Use `{ }` to write JavaScript expressions inside JSX.
- JSX supports expressions, not statements; if, for, and switch cannot be written directly inside JSX.
- Use className instead of class, and htmlFor instead of for.
- JSX attributes use camelCase, like onClick, tabIndex, and backgroundColor.
- Conditional rendering is commonly done using ternary, &&, or helper functions.
- Always close tags properly, including self-closing tags like `<img />` and `<input />`.
- JSX automatically escapes values, which helps prevent XSS when rendering plain text.

## One-Line Final Recall

JSX lets you write UI-like syntax inside JavaScript, but it follows JavaScript rules, not pure HTML rules.
