# Topic: Lists & Keys

- Use `map()` to render lists of elements from arrays.

- Each list item needs a unique `key` so React can track changes efficiently.

- Keys help React identify which items were added, removed, or reordered.

- Use stable IDs from data as keys whenever possible.

- Avoid using array index as key if the list can reorder, insert, or delete items.

- Index keys can cause wrong UI state, input bugs, or incorrect component reuse.

- The `key` prop is only for React; it is not available inside the child component as a normal prop.

- Keep list rendering readable; move complex item UI into a separate component.

- Common mistake: putting the key inside the child component instead of on the element returned by `map()`.

- Real-world use: rendering tables, cards, dropdown options, menus, search results, and dynamic forms.

## One-Line Final Recall

Lists render repeated UI from arrays, and keys help React correctly track each item during updates.
