# Topic: SSR

- SSR means rendering React components on the server and sending HTML to the browser.

- It improves initial page load because users see HTML before JavaScript fully loads.

- SSR is useful for SEO because search engines can read server-rendered content more easily.

- After HTML loads, React hydrates the page to make it interactive.

- Hydration connects React event handlers to the existing server-rendered HTML.

- Common frameworks for SSR include Next.js and Remix.

- SSR is best for public pages, blogs, product pages, landing pages, and SEO-heavy apps.

- Avoid using browser-only APIs like `window` or `localStorage` directly during server render.

- Common mistake: causing hydration mismatch by rendering different HTML on server and client.

- Interview trap: SSR improves first paint and SEO, but JavaScript is still needed for full interactivity.

## One-Line Final Recall

SSR renders React HTML on the server first, then hydrates it on the client to make the app interactive.
