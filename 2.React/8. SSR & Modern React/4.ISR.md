# Topic: ISR

- ISR means Incremental Static Regeneration, mainly used in frameworks like Next.js.

- It allows static pages to be regenerated after build time without rebuilding the whole site.

- ISR gives the speed of SSG with the ability to update stale content.

- Pages are served from cache first, then regenerated in the background based on revalidation rules.

- Best use cases: product pages, blogs, news pages, docs, and content that changes occasionally.

- ISR helps avoid rebuilding thousands of pages every time content changes.

- It is useful when content is mostly static but still needs periodic updates.

- Common mistake: using ISR for highly user-specific or real-time data.

- Interview trap: ISR is not SSR; SSR renders on every request, while ISR regenerates static pages over time.

- Real-world use: e-commerce product pages, CMS-driven sites, marketing pages, and large content websites.

## One-Line Final Recall

ISR updates static pages after build time, giving fast SSG performance with controlled content freshness.
