# Topic: Next.js Architecture

- Next.js is a React framework for routing, rendering, data fetching, and full-stack app structure.

- App Router uses the `app/` directory with file-based routing and nested layouts.

- Pages are built using Server Components by default in the App Router.

- Use `"use client"` only when a component needs state, effects, event handlers, or browser APIs.

- Next.js supports multiple rendering patterns: SSR, SSG, ISR, CSR, and Streaming SSR.

- API routes or route handlers allow backend logic inside the same project.

- Layouts help share UI like navbar, sidebar, and auth shell across routes.

- Data fetching can happen on the server, reducing client-side API calls and bundle size.

- Common mistake: making everything a Client Component, which removes many Next.js performance benefits.

- Interview trap: Next.js is not just routing; it is a rendering and full-stack architecture built on React.

## One-Line Final Recall

Next.js architecture combines React with file-based routing, server rendering, layouts, data fetching, and full-stack capabilities.
