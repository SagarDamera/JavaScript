# Topic: Streaming SSR

- Streaming SSR sends server-rendered HTML to the browser in chunks instead of waiting for the full page.

- It improves perceived performance because users can see parts of the page earlier.

- Works well with `Suspense` boundaries to stream ready sections first and show fallbacks for slower sections.

- Critical UI can be sent first while slower data or components continue loading.

- Hydration can also happen progressively, making parts of the page interactive sooner.

- Best use case: large pages, dashboards, product pages, and content-heavy apps with mixed loading speeds.

- Common frameworks like Next.js and Remix support streaming SSR patterns.

- Common mistake: using one large `Suspense` boundary, which reduces streaming benefits.

- Interview trap: Streaming SSR improves delivery and perceived speed, but slow backend data still needs optimization.

- Real-world relevance: faster first paint, better user experience, and smoother loading for complex server-rendered React apps.

## One-Line Final Recall

Streaming SSR sends HTML in chunks so users can see and interact with ready parts of the page sooner.
