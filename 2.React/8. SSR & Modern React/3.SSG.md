# Topic: SSG

- SSG means generating HTML pages at build time before users request them.

- It gives very fast page loads because prebuilt HTML can be served from CDN.

- SSG is good for SEO because content is already present in the HTML.

- Best use cases: blogs, documentation, marketing pages, product pages, and landing pages.

- Data is fetched during build time, not on every request.

- Content may become stale until the site is rebuilt or regenerated.

- Frameworks like Next.js support SSG using static generation features.

- Use SSG when content does not change frequently for each user.

- Common mistake: using SSG for highly dynamic or user-specific pages.

- Interview trap: SSG is different from SSR; SSR renders per request, SSG renders at build time.

## One-Line Final Recall

SSG prebuilds React pages as static HTML at build time for fast loading and strong SEO.
