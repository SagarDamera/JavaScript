# Topic: CSR

- CSR means React renders the app in the browser using JavaScript.

- The server usually sends a minimal HTML file and JavaScript bundle.

- Browser downloads, parses, and runs JavaScript before the full UI appears.

- CSR works well for dashboards, internal tools, admin apps, and highly interactive apps.

- Initial load can be slower compared to SSR if the JavaScript bundle is large.

- SEO can be harder because content may not exist in the initial HTML.

- After initial load, navigation is usually fast because routing happens on the client.

- Common mistake: using CSR for SEO-heavy public pages without considering SSR or SSG.

- Interview trap: CSR does not mean no server; APIs still provide data from the backend.

- Real-world use: single-page applications, authenticated apps, portals, and complex UI workflows.

## One-Line Final Recall

CSR renders React in the browser using JavaScript, making it great for interactive apps but weaker for initial load and SEO.
