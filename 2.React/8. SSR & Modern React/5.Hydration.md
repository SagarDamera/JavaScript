# Topic: Hydration

- Hydration is the process where React attaches event handlers to server-rendered HTML.

- It makes static HTML from SSR or SSG interactive in the browser.

- React reuses existing HTML instead of creating the page from scratch.

- Hydration happens after JavaScript loads on the client.

- Common use case: Next.js or Remix apps using SSR/SSG.

- Hydration mismatch happens when server HTML and client render output are different.

- Avoid using browser-only values like `window`, `localStorage`, or random values directly during initial render.

- Use `useEffect` for client-only logic that should run after hydration.

- Common mistake: rendering different content on server and client, causing warnings or broken UI.

- Interview trap: SSR shows the page faster, but hydration is needed before users can fully interact.

## One-Line Final Recall

Hydration connects React’s client-side behavior to server-rendered HTML so the page becomes interactive.
