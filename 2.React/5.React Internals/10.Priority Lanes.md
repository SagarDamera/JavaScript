# Topic: Priority Lanes

- Priority lanes are React’s internal system for assigning priority to different updates.

- They help React decide which updates should be processed first.

- Urgent updates like typing, clicking, and input changes get higher priority lanes.

- Non-urgent updates like filtering large lists or rendering background UI get lower priority lanes.

- Multiple updates can be grouped into lanes based on their priority and timing.

- React can pause lower-priority work to handle higher-priority updates first.

- Priority lanes work with Fiber and Scheduler to support concurrent rendering.

- Developers do not use lanes directly; React manages them internally.

- Interview trap: lanes are not browser threads; they are React’s priority model for rendering work.

- Real-world relevance: lanes help React keep the UI responsive during heavy updates.

## One-Line Final Recall

Priority lanes let React organize updates by urgency so important user interactions are handled first.
