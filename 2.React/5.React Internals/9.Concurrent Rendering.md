# Topic: Concurrent Rendering

- Concurrent rendering lets React prepare UI updates without blocking urgent user interactions.

- React can pause, resume, restart, or discard rendering work before committing it.

- It helps keep the app responsive during expensive UI updates.

- Urgent updates like typing and clicking get higher priority than non-urgent updates.

- Concurrent rendering is enabled through React 18 features like transitions and Suspense.

- Render phase can be interrupted, but commit phase is always completed once started.

- Components must stay pure because render work may run more than once.

- Common mistake: assuming concurrent rendering means multiple renders happen at the same time.

- Interview trap: concurrent rendering improves responsiveness, not necessarily raw rendering speed.

- Real-world use: search filtering, large lists, dashboards, slow tabs, and async UI transitions.

## One-Line Final Recall

Concurrent rendering lets React prioritize urgent updates by interrupting and scheduling render work before committing UI changes.
