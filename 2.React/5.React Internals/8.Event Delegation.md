# Topic: Event Delegation

- Event delegation means handling events at a higher level instead of attaching listeners to every child element.

- React uses a synthetic event system and delegates most events through the root container.

- This improves performance because React does not need separate native listeners for every element.

- Event handlers still look like normal props, such as `onClick`, `onChange`, and `onSubmit`.

- React normalizes events across browsers using Synthetic Events.

- Event bubbling allows child events to travel up to parent components.

- Use `event.stopPropagation()` only when you intentionally want to stop bubbling.

- Common mistake: overusing `stopPropagation`, which can break parent-level handlers.

- Interview trap: React event handling feels per-element, but internally React manages events efficiently using delegation.

- Real-world use: handling clicks in lists, tables, menus, dropdowns, modals, and parent-child UI interactions.

## One-Line Final Recall

Event delegation lets React handle many child events efficiently through higher-level event management.
