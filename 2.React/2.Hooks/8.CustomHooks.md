# Topic: Custom Hooks

- Custom hooks are reusable functions that contain shared React hook logic.

- Name must start with `use`, like `useFetch`, `useAuth`, or `useDebounce`.

- Use custom hooks to avoid repeating state, effects, or logic across components.

- Custom hooks can use built-in hooks like `useState`, `useEffect`, `useRef`, and `useMemo`.

- They do not share state automatically; each component using the hook gets its own state.

- Keep custom hooks focused on one responsibility.

- Return only what the component needs, like data, loading, error, or handler functions.

- Common mistake: putting UI JSX inside custom hooks; hooks should manage logic, not render UI.

- Interview trap: custom hooks follow the same Rules of Hooks as normal hooks.

- Real-world use: API fetching, form logic, auth state, debouncing, pagination, and local storage sync.

## One-Line Final Recall

Custom hooks let you reuse React stateful logic across components without repeating code.

---

### 1. What problem do custom hooks solve in React?

Custom hooks let you reuse stateful logic across multiple components.  
They help keep components clean by moving reusable logic like fetching, forms, timers, or subscriptions into separate functions.

```jsx
function useCounter() {
  const [count, setCount] = useState(0);

  return { count, increment: () => setCount((c) => c + 1) };
}
```

Tip: Custom hooks reuse logic, not UI.

---

### 2. How is a custom hook different from a regular JavaScript function?

A custom hook can call React hooks like `useState`, `useEffect`, or `useReducer`.  
A regular JavaScript function cannot safely use hooks unless it follows hook rules and is used as a hook.

```jsx
function useUser() {
  const [user, setUser] = useState(null);
  return { user, setUser };
}
```

Tip: A custom hook is a function that uses React hook behavior.

---

### 3. Why must a custom hook name start with `use`?

React and ESLint use the `use` prefix to recognize hook functions.  
This helps enforce the Rules of Hooks, like calling hooks only at the top level.

```jsx
function useWindowSize() {
  const [size, setSize] = useState(window.innerWidth);
  return size;
}
```

Tip: The `use` prefix helps React tooling detect hook usage correctly.

---

### 4. What React hooks can be used inside a custom hook?

A custom hook can use any built-in React hook or another custom hook.  
For example, it can use `useState`, `useEffect`, `useMemo`, `useCallback`, `useReducer`, or `useContext`.

```jsx
function useAuth() {
  const user = useContext(AuthContext);

  const isLoggedIn = useMemo(() => {
    return Boolean(user);
  }, [user]);

  return { user, isLoggedIn };
}
```

Tip: Custom hooks can compose multiple hooks together.

---

### 5. What can a custom hook return?

A custom hook can return anything: state, functions, objects, arrays, refs, or computed values.  
The return shape should be simple and easy for components to use.

```jsx
function useToggle(initialValue = false) {
  const [value, setValue] = useState(initialValue);

  const toggle = () => setValue((v) => !v);

  return { value, toggle, setValue };
}
```

Tip: Return the smallest API the component needs.

---

### 6. Can custom hooks share state between components automatically?

No, custom hooks do not automatically share state between components.  
Each component that calls the custom hook gets its own separate hook state.

```jsx
function useCounter() {
  const [count, setCount] = useState(0);
  return { count, setCount };
}
```

Tip: Custom hooks share logic, not state.

---

### 7. How does each component get its own isolated state when using the same custom hook?

React stores hook state separately for each component Fiber.  
So when two components call the same custom hook, each call belongs to that component’s own render state.

```jsx
function ComponentA() {
  const counter = useCounter();
}

function ComponentB() {
  const counter = useCounter();
}
```

Tip: Same custom hook function, different component state.

---

### 8. When should you extract logic into a custom hook?

Extract logic into a custom hook when multiple components need the same stateful behavior.  
It is also useful when a component becomes too large because of effects, subscriptions, or reusable business logic.

```jsx
function useDocumentTitle(title) {
  useEffect(() => {
    document.title = title;
  }, [title]);
}
```

Tip: Create a custom hook when logic is reusable or improves readability.

---

### 9. When should you avoid creating a custom hook?

Avoid creating a custom hook for logic used only once or logic that is simple enough to stay inside the component.  
Too many small hooks can make code harder to follow.

```jsx
// This may not need a custom hook
const fullName = `${firstName} ${lastName}`;
```

Tip: Do not extract too early; custom hooks should make code clearer.

---

### 10. How do dependency arrays inside a custom hook affect the component using it?

Dependency arrays inside a custom hook control when its internal effects, memos, or callbacks update.  
If dependencies are wrong, the component using the hook may see stale values or unnecessary re-runs.

```jsx
function useUser(userId) {
  useEffect(() => {
    fetchUser(userId);
  }, [userId]);
}
```

Tip: Custom hook dependencies still follow the same hook rules.

---

### 11. How can stale closures happen inside a custom hook?

Stale closures happen when a hook callback or effect captures old props, state, or arguments.  
This usually happens because a changing value is missing from the dependency array.

```jsx
function useLogger(value) {
  useEffect(() => {
    console.log(value);
  }, []); // missing value
}
```

Tip: Custom hooks can have stale closures the same way components can.

---

### 12. How do you design a custom hook API so it is reusable and easy to understand?

A good custom hook should have clear inputs and a simple return value.  
Avoid exposing too many internal details; return only what the component needs.

```jsx
function useToggle(initialValue = false) {
  const [value, setValue] = useState(initialValue);

  const toggle = () => setValue((v) => !v);

  return { value, toggle, setValue };
}
```

Tip: Design custom hooks like small reusable APIs.

---

### 13. Why should custom hooks avoid hiding too much complex behavior?

If a custom hook hides too much logic, components become hard to debug and behavior becomes unclear.  
A hook should improve readability, not make important side effects invisible.

```jsx
function useDashboardData() {
  // Fetching, polling, permissions, analytics, and routing here
  // can become too much hidden behavior.
}
```

Tip: Keep custom hooks focused on one clear responsibility.

---

### 14. How can custom hooks improve testing and code organization?

Custom hooks move reusable stateful logic out of components.  
This makes components smaller and allows the hook logic to be tested separately.

```jsx
function useForm(initialValues) {
  const [values, setValues] = useState(initialValues);

  return { values, setValues };
}
```

Tip: Custom hooks make logic easier to reuse, test, and maintain.

---

### 15. Why can’t a custom hook be called conditionally inside a component?

Custom hooks can contain React hooks internally, so they must follow the Rules of Hooks.  
Calling them conditionally can change hook order between renders and break React’s hook state matching.

```jsx
// Wrong
if (isLoggedIn) {
  const user = useAuth();
}

// Correct
const user = useAuth();
```

Tip: Always call custom hooks at the top level of the component.

### 16. How would you build a `useFetch` custom hook for API loading, success, and error states?

Create internal state for `data`, `loading`, and `error`.  
Run the API call inside `useEffect`, then update state based on success or failure.

```jsx
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    let ignore = false;

    setLoading(true);
    setError(null);

    fetch(url)
      .then((res) => res.json())
      .then((result) => {
        if (!ignore) setData(result);
      })
      .catch((err) => {
        if (!ignore) setError(err.message);
      })
      .finally(() => {
        if (!ignore) setLoading(false);
      });

    return () => {
      ignore = true;
    };
  }, [url]);

  return { data, loading, error };
}
```

Tip: A good `useFetch` should handle loading, error, success, and outdated responses.

### 17. How would you build a `useDebounce` custom hook for search input optimization?

Store a delayed version of the input value.  
Update it only after the value stops changing for the given delay.

```jsx
function useDebounce(value, delay = 500) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const timerId = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(timerId);
    };
  }, [value, delay]);

  return debouncedValue;
}
```

Tip: `useDebounce` is useful for search boxes, filters, and API calls.

### 18. How would you build a `useLocalStorage` custom hook to persist state in browser storage?

Initialize state from `localStorage`, then update both React state and `localStorage` together.  
Handle JSON parsing safely because stored values are strings.

```jsx
function useLocalStorage(key, initialValue) {
  const [value, setValue] = useState(() => {
    const savedValue = localStorage.getItem(key);
    return savedValue ? JSON.parse(savedValue) : initialValue;
  });

  const updateValue = (newValue) => {
    setValue(newValue);
    localStorage.setItem(key, JSON.stringify(newValue));
  };

  return [value, updateValue];
}
```

Tip: `useLocalStorage` keeps UI state persistent after page refresh.

### 19. How would you build a `useWindowSize` custom hook to track browser resize events?

Store window width and height in state.  
Add a resize event listener in `useEffect` and clean it up when the component unmounts.

```jsx
function useWindowSize() {
  const [size, setSize] = useState({
    width: window.innerWidth,
    height: window.innerHeight,
  });

  useEffect(() => {
    const handleResize = () => {
      setSize({
        width: window.innerWidth,
        height: window.innerHeight,
      });
    };

    window.addEventListener("resize", handleResize);

    return () => {
      window.removeEventListener("resize", handleResize);
    };
  }, []);

  return size;
}
```

Tip: Always clean up event listeners inside custom hooks.

### 20. How would you build a `useOnlineStatus` custom hook to detect internet connection changes?

Use `navigator.onLine` as the initial value.  
Listen to browser `online` and `offline` events to update the status.

```jsx
function useOnlineStatus() {
  const [isOnline, setIsOnline] = useState(navigator.onLine);

  useEffect(() => {
    const handleOnline = () => setIsOnline(true);
    const handleOffline = () => setIsOnline(false);

    window.addEventListener("online", handleOnline);
    window.addEventListener("offline", handleOffline);

    return () => {
      window.removeEventListener("online", handleOnline);
      window.removeEventListener("offline", handleOffline);
    };
  }, []);

  return isOnline;
}
```

Tip: `useOnlineStatus` is useful for offline banners and retry behavior.

### 21. How would you build a `useClickOutside` custom hook to close a dropdown or modal?

Create a `ref` for the target element and listen for document clicks.  
If the click target is outside the referenced element, call the provided close handler.

```jsx
function useClickOutside(onClose) {
  const ref = useRef(null);

  useEffect(() => {
    const handleClick = (event) => {
      if (ref.current && !ref.current.contains(event.target)) {
        onClose();
      }
    };

    document.addEventListener("mousedown", handleClick);

    return () => {
      document.removeEventListener("mousedown", handleClick);
    };
  }, [onClose]);

  return ref;
}
```

Tip: `useClickOutside` is useful for dropdowns, modals, popovers, and menus.

### 22. How would you build a `usePrevious` custom hook to compare previous and current values?

Use a `ref` to store the previous value because updating a ref does not cause re-render.  
Update the ref inside `useEffect` after each render.

```jsx
function usePrevious(value) {
  const previousRef = useRef();

  useEffect(() => {
    previousRef.current = value;
  }, [value]);

  return previousRef.current;
}
```

Tip: `usePrevious` uses `useRef` because refs persist across renders.

### 23. How would you build a `useForm` custom hook to manage form values, validation, and submit state?

Store form values, errors, and submit state inside the hook.  
Expose handlers like `handleChange`, `handleSubmit`, and `reset` so components stay clean.

```jsx
function useForm(initialValues, validate, onSubmit) {
  const [values, setValues] = useState(initialValues);
  const [errors, setErrors] = useState({});
  const [isSubmitting, setIsSubmitting] = useState(false);

  const handleChange = (event) => {
    const { name, value } = event.target;
    setValues((prev) => ({ ...prev, [name]: value }));
  };

  const handleSubmit = async (event) => {
    event.preventDefault();

    const validationErrors = validate(values);
    setErrors(validationErrors);

    if (Object.keys(validationErrors).length === 0) {
      setIsSubmitting(true);
      await onSubmit(values);
      setIsSubmitting(false);
    }
  };

  return { values, errors, isSubmitting, handleChange, handleSubmit };
}
```

Tip: A good `useForm` separates form logic from form UI.

### 24. How would you build a `useAuth` custom hook to expose logged-in user, login, and logout behavior?

Read auth state from context or local state and return user-related actions.  
The hook should expose a small API like `user`, `login`, `logout`, and `isAuthenticated`.

```jsx
function useAuth() {
  const { user, setUser } = useContext(AuthContext);

  const login = async (credentials) => {
    const loggedInUser = await loginApi(credentials);
    setUser(loggedInUser);
  };

  const logout = () => {
    clearToken();
    setUser(null);
  };

  return {
    user,
    login,
    logout,
    isAuthenticated: Boolean(user),
  };
}
```

Tip: `useAuth` hides auth details and gives components a clean auth API.

### 25. How would you build a `useInfiniteScroll` custom hook for loading more data when the user reaches the bottom?

Use an `IntersectionObserver` to detect when a target element becomes visible.  
When the target is visible and more data exists, call the load-more function.

```jsx
function useInfiniteScroll({ hasMore, loading, onLoadMore }) {
  const loaderRef = useRef(null);

  useEffect(() => {
    if (!hasMore || loading) return;

    const observer = new IntersectionObserver(([entry]) => {
      if (entry.isIntersecting) {
        onLoadMore();
      }
    });

    const currentLoader = loaderRef.current;

    if (currentLoader) {
      observer.observe(currentLoader);
    }

    return () => {
      if (currentLoader) {
        observer.unobserve(currentLoader);
      }
    };
  }, [hasMore, loading, onLoadMore]);

  return loaderRef;
}
```

Tip: `useInfiniteScroll` should avoid duplicate loads while data is already loading.
