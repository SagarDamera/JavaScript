# Topic: useMemo

- `useMemo` memoizes a calculated value between renders.

- Use it when an expensive calculation should not run on every render.

- It recalculates only when one of its dependencies changes.

- Good use case: filtering, sorting, grouping, or transforming large data.

- Do not use `useMemo` for every small calculation; it adds extra complexity.

- `useMemo` improves performance only when the calculation or re-render cost is meaningful.

- Dependencies must be correct; missing dependencies can create stale values.

- Common mistake: using `useMemo` to hide bad state structure or unnecessary parent renders.

- Interview trap: `useMemo` does not memoize components; it memoizes values.

- Real-world use: optimized table filters, search results, derived data, and expensive computed props.

## One-Line Final Recall

`useMemo` caches expensive computed values and recalculates them only when dependencies change.

---

### 1. What problem does `useMemo` solve in React?

`useMemo` helps avoid recalculating expensive derived values on every render.  
It is mainly used when a calculation is costly or when a stable object/array reference is needed.

```jsx
const filteredUsers = useMemo(() => {
  return users.filter((user) => user.active);
}, [users]);
```

Tip: `useMemo` optimizes recalculation, not rendering itself.

---

### 2. How does `useMemo` differ from calculating a value directly during render?

A normal calculation runs every time the component renders.  
`useMemo` runs the calculation only when one of its dependencies changes.

```jsx
// Runs on every render
const total = items.reduce((sum, item) => sum + item.price, 0);

// Runs only when items changes
const total = useMemo(() => {
  return items.reduce((sum, item) => sum + item.price, 0);
}, [items]);
```

Tip: Direct calculation is fine unless it is expensive or causes unstable references.

---

### 3. When should you use `useMemo` in a real React application?

Use `useMemo` when deriving expensive values like filtered, sorted, or transformed data.  
It is also useful when passing objects or arrays to memoized child components.

```jsx
const sortedProducts = useMemo(() => {
  return [...products].sort((a, b) => a.price - b.price);
}, [products]);
```

Tip: Use `useMemo` only when there is a real performance or reference-stability need.

---

### 4. What does `useMemo` return?

`useMemo` returns the value produced by the callback function.  
That value can be anything like a number, string, object, array, JSX, or computed data.

```jsx
const userInfo = useMemo(() => {
  return {
    name: user.name,
    role: user.role,
  };
}, [user.name, user.role]);
```

Tip: `useMemo` stores and returns a computed value between renders.

---

### 5. How does the dependency array control when `useMemo` recalculates its value?

React compares the dependency values between renders.  
If any dependency changes, React runs the `useMemo` callback again and stores the new value.

```jsx
const discountedPrice = useMemo(() => {
  return price - price * discount;
}, [price, discount]);
```

Tip: `useMemo` recalculates only when one of its dependencies changes.

---

### 6. What happens if the dependency array is empty in `useMemo`?

If the dependency array is empty, `useMemo` runs only during the initial render.  
After that, React reuses the same memoized value for the component’s lifetime.

```jsx
const config = useMemo(() => {
  return { theme: "dark", pageSize: 20 };
}, []);
```

Tip: Empty dependency array means “calculate once for this component instance.”

---

### 7. What happens if you omit the dependency array in `useMemo`?

If you omit the dependency array, React recalculates the value on every render.  
In that case, `useMemo` gives almost no caching benefit.

```jsx
const total = useMemo(() => {
  return items.reduce((sum, item) => sum + item.price, 0);
});
```

Tip: Without dependencies, `useMemo` runs every render like a normal calculation.

---

### 8. Can `useMemo` prevent a component from re-rendering?

No, `useMemo` cannot stop the current component from re-rendering.  
It only memoizes a calculated value inside that render.

```jsx
const expensiveValue = useMemo(() => {
  return calculateExpensiveValue(count);
}, [count]);
```

Tip: `useMemo` saves recalculation, but it does not block renders.

---

### 9. Why is `useMemo` not a guarantee that React will always keep the cached value?

React treats `useMemo` as a performance optimization, not a semantic guarantee.  
React may discard memoized values in some cases and recalculate them later.

```jsx
const value = useMemo(() => {
  return computeValue(data);
}, [data]);
```

Tip: Your code should still work correctly even if `useMemo` recalculates.

---

### 10. How can incorrect dependencies in `useMemo` cause stale values?

If a value used inside `useMemo` is missing from the dependency array, React may reuse an old result.  
This causes the memoized value to be based on outdated data.

```jsx
const message = useMemo(() => {
  return `${user.name} has ${count} items`;
}, [user.name]); // missing count
```

Tip: Every changing value used inside `useMemo` should be included in dependencies.

---

### 11. Why can adding unnecessary dependencies to `useMemo` reduce its benefit?

`useMemo` recalculates whenever any dependency changes.  
If you add values that are not actually used in the calculation, the memoized value may be recreated more often than needed.

```jsx
const total = useMemo(() => {
  return items.reduce((sum, item) => sum + item.price, 0);
}, [items, theme]); // theme is unnecessary
```

Tip: Add only the values that are used inside the `useMemo` calculation.

---

### 12. How does `useMemo` help when passing derived objects or arrays to memoized child components?

Objects and arrays created during render get a new reference every time.  
`useMemo` keeps the same reference when dependencies do not change, helping `React.memo` child components avoid unnecessary re-renders.

```jsx
const activeUsers = useMemo(() => {
  return users.filter((user) => user.active);
}, [users]);

return <UserList users={activeUsers} />;
```

Tip: `useMemo` is useful when reference stability matters.

---

### 13. What is the difference between `useMemo` and `React.memo`?

`useMemo` memoizes a calculated value inside a component.  
`React.memo` memoizes a component and skips re-rendering it when props are the same.

```jsx
const value = useMemo(() => calculateValue(data), [data]);

const UserList = React.memo(function UserList({ users }) {
  return <div>{users.length}</div>;
});
```

Tip: `useMemo` is for values, `React.memo` is for components.

---

### 14. What is the difference between `useMemo` and `useCallback`?

`useMemo` memoizes the result of a function call.  
`useCallback` memoizes the function itself so its reference stays stable.

```jsx
const value = useMemo(() => {
  return count * 2;
}, [count]);

const handleClick = useCallback(() => {
  console.log(count);
}, [count]);
```

Tip: `useCallback(fn, deps)` is similar to `useMemo(() => fn, deps)`.

---

### 15. Why should side effects not be placed inside `useMemo`?

`useMemo` runs during render, and render should stay pure.  
Side effects like API calls, subscriptions, timers, or state updates should be handled in `useEffect`.

```jsx
// Wrong
const value = useMemo(() => {
  localStorage.setItem("count", count);
  return count * 2;
}, [count]);

// Correct
useEffect(() => {
  localStorage.setItem("count", count);
}, [count]);
```

Tip: Use `useMemo` for calculations, not side effects.

---

### 16. Can `useMemo` be used for async operations? Why or why not?

`useMemo` should not be used to perform async operations like API calls.  
It runs during render, and render should stay pure and synchronous.

```jsx
// Wrong
const data = useMemo(async () => {
  return await fetchUsers();
}, []);

// Correct
useEffect(() => {
  fetchUsers().then(setUsers);
}, []);
```

Tip: Use `useEffect` for async side effects, not `useMemo`.

---

### 17. How can `useMemo` be misused for cheap calculations?

Using `useMemo` for simple calculations can make the code more complex without improving performance.  
For cheap operations, recalculating directly during render is usually better.

```jsx
// Unnecessary
const fullName = useMemo(() => {
  return `${firstName} ${lastName}`;
}, [firstName, lastName]);

// Better
const fullName = `${firstName} ${lastName}`;
```

Tip: Do not memoize everything; memoize only when there is a clear benefit.

---

### 18. What are the performance costs of using `useMemo` itself?

`useMemo` has its own cost because React must store the value and compare dependencies on every render.  
If the calculation is cheap, this overhead may be greater than the calculation itself.

```jsx
const doubled = useMemo(() => {
  return count * 2;
}, [count]);
```

Tip: `useMemo` is useful only when the saved work is greater than the memoization cost.

---

### 19. How does `useMemo` behave during the initial render?

During the initial render, React always runs the `useMemo` callback.  
The returned value is stored and reused on later renders if dependencies do not change.

```jsx
const expensiveValue = useMemo(() => {
  console.log("Calculating...");
  return calculateExpensiveValue(input);
}, [input]);
```

Output:

```txt
Calculating...
```

Tip: `useMemo` does not skip the first calculation.

---

### 20. How does `useMemo` behave when a parent component re-renders but dependencies do not change?

The child component may still render, but the `useMemo` value is reused if dependencies are unchanged.  
React does not rerun the memo callback in that case.

```jsx
const filteredItems = useMemo(() => {
  console.log("Filtering...");
  return items.filter((item) => item.active);
}, [items]);
```

Output if parent re-renders but `items` is same:

```txt
// No "Filtering..." log
```

Tip: Parent re-render does not automatically recalculate `useMemo`.

---

### 21. What happens if a dependency is an object or array created inline during render?

A new object or array reference is created on every render.  
React sees it as changed, so `useMemo` recalculates every time.

```jsx
const options = { sortBy: "price" };

const sortedItems = useMemo(() => {
  return sortItems(items, options);
}, [items, options]);
```

Tip: Inline objects/arrays break memoization because their reference changes every render.

---

### 22. How can `useMemo` help avoid expensive filtering, sorting, or transformation on every render?

`useMemo` stores the result of expensive data transformations.  
React recalculates the value only when the source data or related dependencies change.

```jsx
const visibleProducts = useMemo(() => {
  return products
    .filter((product) => product.inStock)
    .sort((a, b) => a.price - b.price);
}, [products]);
```

Tip: `useMemo` is useful for expensive derived data from props or state.

---

### 23. How does React compare dependencies inside `useMemo`?

React compares each dependency with its previous value using `Object.is`.  
For primitives, value comparison works as expected; for objects/functions/arrays, reference must be the same.

```jsx
const value = useMemo(() => {
  return calculateTotal(items);
}, [items]);
```

Tip: Same reference means unchanged; new reference means changed.

---

### 24. How does React internally associate a `useMemo` call with a component Fiber across renders?

React stores hook data as a linked list on the component’s Fiber.  
On each render, React matches hooks by call order, so the same `useMemo` position gets the previous memoized value and dependencies.

```jsx
const total = useMemo(() => calculateTotal(items), [items]);
const count = useMemo(() => items.length, [items]);
```

Tip: React identifies hooks by order, not by variable name.

---

### 25. Why can’t `useMemo` be called conditionally inside a component?

Hooks must be called in the same order on every render.  
Calling `useMemo` conditionally changes the hook order and breaks React’s ability to match previous hook state.

```jsx
// Wrong
if (isAdmin) {
  const permissions = useMemo(() => getPermissions(user), [user]);
}

// Correct
const permissions = useMemo(() => {
  return isAdmin ? getPermissions(user) : [];
}, [isAdmin, user]);
```

Tip: Keep `useMemo` at the top level of the component, never inside conditions or loops.
