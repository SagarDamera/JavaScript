# Topic: Hooks Rules

- Hooks must be called only at the top level of a React function component or custom hook.

- Do not call hooks inside loops, conditions, nested functions, or `try/catch`.

- Hooks must run in the same order on every render so React can track state correctly.

- Hooks can only be used in functional components or custom hooks, not normal JavaScript functions.

- Custom hook names must start with `use` so React and lint rules can detect hook usage.

- Use dependency arrays correctly in hooks like `useEffect`, `useMemo`, and `useCallback`.

- Missing dependencies can cause stale values; extra unstable dependencies can cause unnecessary reruns.

- Common mistake: calling a hook after an early return condition.

- Interview trap: hooks depend on call order, not variable names.

- Real-world relevance: following hook rules prevents unpredictable state bugs and effect issues.

## One-Line Final Recall

Hooks must be called consistently at the top level so React can track state and effects correctly.

---

### 1. What are the main Rules of Hooks in React?

Hooks must be called only at the top level of React function components or custom hooks.  
They should not be called inside conditions, loops, nested functions, event handlers, or regular JavaScript functions.

```jsx
function Profile() {
  const [user, setUser] = useState(null);
  useEffect(() => {}, []);

  return <div>Profile</div>;
}
```

Tip: Hooks must run in the same order on every render.

---

### 2. Why must hooks be called only at the top level of a component or custom hook?

React tracks hooks by their call order during render.  
Calling hooks at the top level keeps the order stable across every render.

```jsx
function Counter() {
  const [count, setCount] = useState(0);
  const doubled = useMemo(() => count * 2, [count]);

  return <div>{doubled}</div>;
}
```

Tip: Top-level hook calls help React match each hook with its previous state.

---

### 3. Why can’t hooks be called inside conditions?

A condition may run on one render and skip on another render.  
That changes the hook order, so React may attach the wrong state or effect to the wrong hook.

```jsx
// Wrong
if (isLoggedIn) {
  const user = useContext(AuthContext);
}

// Correct
const user = useContext(AuthContext);

if (!isLoggedIn) {
  return null;
}
```

Tip: Put conditions inside the hook logic, not around the hook call.

---

### 4. Why can’t hooks be called inside loops?

A loop can run a different number of times on different renders.  
That means React cannot guarantee the same hook order and count each time.

```jsx
// Wrong
items.forEach((item) => {
  const value = useMemo(() => item.name, [item]);
});

// Correct
const names = useMemo(() => {
  return items.map((item) => item.name);
}, [items]);
```

Tip: Do not create hook calls dynamically inside loops.

---

### 5. Why can’t hooks be called inside nested functions?

Nested functions may not run during every render, or they may run later outside React’s render flow.  
React needs hooks to run directly during component rendering in a predictable order.

```jsx
// Wrong
function Profile() {
  function loadUser() {
    const user = useContext(AuthContext);
  }
}

// Correct
function Profile() {
  const user = useContext(AuthContext);

  function loadUser() {
    console.log(user);
  }
}
```

Tip: Hooks belong in the component body or custom hook body, not inside nested functions.

---

### 6. Why can hooks only be called from React function components or custom hooks?

Hooks need React’s render context to know which component Fiber they belong to.  
Regular JavaScript functions do not have that React hook tracking context.

```jsx
// Wrong
function getUser() {
  const user = useContext(UserContext);
  return user;
}

// Correct
function useUser() {
  const user = useContext(UserContext);
  return user;
}
```

Tip: Hooks must run where React can track component state.

---

### 7. Why must custom hook names start with `use`?

The `use` prefix tells React tooling and ESLint that the function is a hook.  
This helps enforce the Rules of Hooks and catch invalid hook calls.

```jsx
function useWindowSize() {
  const [size, setSize] = useState(window.innerWidth);
  return size;
}
```

Tip: Custom hooks should always start with `use`.

---

### 8. How does React use hook call order to preserve hook state across renders?

React stores hooks in order on the component Fiber.  
On the next render, React walks through hooks in the same order and gives each hook its previous state.

```jsx
const [count, setCount] = useState(0);
const user = useContext(UserContext);
const total = useMemo(() => count * 2, [count]);
```

Tip: React matches hooks by position, not by variable name.

---

### 9. What happens internally if hook order changes between renders?

React may read the wrong previous hook data for the current hook call.  
This can cause incorrect state, broken effects, or runtime hook order errors.

```jsx
// Wrong
if (showDetails) {
  useEffect(() => {}, []);
}

const [count, setCount] = useState(0);
```

Tip: Changing hook order breaks React’s hook matching system.

---

### 10. What error can happen when a component renders fewer hooks than expected?

React may throw an error like “Rendered fewer hooks than expected.”  
This often happens when a component returns early before calling all hooks from the previous render.

```jsx
function Profile({ user }) {
  const [count, setCount] = useState(0);

  if (!user) {
    return null;
  }

  const theme = useContext(ThemeContext);
  return <div>{theme}</div>;
}
```

Tip: Place all hooks before conditional returns.

---

### 11. What error can happen when a component renders more hooks than expected?

React may throw an error like “Rendered more hooks than during the previous render.”  
This usually happens when a hook is called conditionally only on some renders.

```jsx
function Profile({ showDetails }) {
  const [count, setCount] = useState(0);

  if (showDetails) {
    const theme = useContext(ThemeContext); // Wrong
  }

  return <div>{count}</div>;
}
```

Tip: Every render must call the same hooks in the same order.

---

### 12. Why is calling hooks after an early return a common mistake?

If the component returns before reaching some hooks, those hooks are skipped on that render.  
On another render, React may reach them, causing hook order mismatch.

```jsx
function Profile({ user }) {
  if (!user) {
    return null;
  }

  const [count, setCount] = useState(0); // Wrong
  return <div>{count}</div>;
}
```

Tip: Call hooks before early returns.

---

### 13. How should you handle conditional logic when a hook should only run sometimes?

Always call the hook, but put the condition inside the hook logic.  
This keeps hook order stable while still controlling when the logic runs.

```jsx
const user = useContext(UserContext);

useEffect(() => {
  if (!user) return;

  fetchUserDetails(user.id);
}, [user]);
```

Tip: Condition the behavior, not the hook call.

---

### 14. How can you conditionally run logic inside `useEffect` without conditionally calling the hook?

Call `useEffect` every render, then return early inside the effect callback when the condition is false.  
This keeps the hook call order unchanged.

```jsx
useEffect(() => {
  if (!isOpen) return;

  document.title = "Modal Open";

  return () => {
    document.title = "App";
  };
}, [isOpen]);
```

Tip: Put `if` checks inside `useEffect`, not around it.

---

### 15. Why is it invalid to call hooks inside event handlers?

Event handlers run later, outside React’s render phase.  
React cannot track hook order or attach hook state correctly from inside an event handler.

```jsx
// Wrong
function handleClick() {
  const [count, setCount] = useState(0);
}

// Correct
const [count, setCount] = useState(0);

function handleClick() {
  setCount((c) => c + 1);
}
```

Tip: Hooks run during render; event handlers only use hook values or setters.

---

### 16. Why is it invalid to call hooks inside class components?

Hooks are designed for React function components and custom hooks.  
Class components use lifecycle methods and `this.state`, so React does not run hook tracking for them.

```jsx
// Wrong
class Profile extends React.Component {
  render() {
    const user = useContext(UserContext);
    return <div>{user.name}</div>;
  }
}
```

Tip: Use hooks in function components, not class components.

---

### 17. Can hooks be called inside regular JavaScript utility functions?

No, hooks cannot be called inside normal utility functions.  
A function must be a React component or a custom hook starting with `use` to safely call hooks.

```jsx
// Wrong
function getTheme() {
  return useContext(ThemeContext);
}

// Correct
function useTheme() {
  return useContext(ThemeContext);
}
```

Tip: Regular helper functions can receive hook values as arguments, but should not call hooks.

---

### 18. What is the difference between a custom hook and a normal helper function?

A custom hook can call React hooks and participate in React’s hook system.  
A normal helper function is just JavaScript logic and should not call hooks.

```jsx
function useUserName() {
  const user = useContext(UserContext);
  return user.name;
}

function formatUserName(name) {
  return name.trim().toUpperCase();
}
```

Tip: Custom hooks manage React logic; helpers manage plain JavaScript logic.

---

### 19. How does the ESLint Rules of Hooks plugin help catch hook mistakes?

The ESLint plugin detects invalid hook usage before runtime.  
It warns when hooks are called conditionally, inside loops, nested functions, or when dependencies are missing.

```jsx
// ESLint warning
if (isLoggedIn) {
  useEffect(() => {}, []);
}
```

Tip: The Hooks ESLint plugin helps prevent common hook order and dependency bugs.

---

### 20. Why should dependency arrays include all reactive values used inside hooks?

Dependency arrays tell React when to rerun effects, memos, or callbacks.  
If a reactive value is missing, the hook may use stale props, state, or variables from an old render.

```jsx
useEffect(() => {
  console.log(user.name);
}, []); // missing user.name
```

Tip: Include every changing value used inside the hook callback.

---

### 21. What are reactive values in the context of hook dependency arrays?

Reactive values are values that can change between renders.  
They include props, state, variables, and functions declared inside the component body.

```jsx
function Profile({ userId }) {
  const [count, setCount] = useState(0);

  useEffect(() => {
    console.log(userId, count);
  }, [userId, count]);
}
```

Tip: If a value can change across renders and is used inside a hook, it is usually reactive.

---

### 22. Why can missing dependencies cause stale closures?

Hooks capture values from the render where the callback was created.  
If a changing value is missing from the dependency array, the hook may keep using an old value.

```jsx
useEffect(() => {
  console.log(searchText);
}, []); // missing searchText
```

Tip: Missing dependencies often create old-data bugs.

---

### 23. Why can unnecessary dependencies cause repeated executions or unstable references?

A hook reruns when any dependency changes.  
If you include values that are not needed, effects may run too often and memos or callbacks may be recreated unnecessarily.

```jsx
useEffect(() => {
  fetchUser(userId);
}, [userId, theme]); // theme is unnecessary
```

Tip: Extra dependencies can cause avoidable reruns.

---

### 24. How do Rules of Hooks apply inside custom hooks?

Custom hooks can call other hooks, so they must follow the same Rules of Hooks.  
Hooks inside custom hooks should not be called conditionally, inside loops, or inside nested functions.

```jsx
function useUser(isEnabled) {
  useEffect(() => {
    if (!isEnabled) return;

    fetchUser();
  }, [isEnabled]);
}
```

Tip: Custom hooks must also keep hook order stable.

---

### 25. How are the Rules of Hooks connected to React Fiber internals?

React stores hook state in order on the component Fiber.  
During each render, React walks that hook list in the same order to match each hook call with its previous state.

```jsx
const [count, setCount] = useState(0);
const value = useMemo(() => count * 2, [count]);
useEffect(() => {}, [value]);
```

Tip: Hook rules exist because React tracks hooks by call order, not by name.
