# Topic: Tearing

- Tearing happens when different parts of the UI show inconsistent versions of the same external data.

- It can occur during concurrent rendering if external store data changes while React is rendering.

- Example: one component shows old store value while another shows the new value in the same UI frame.

- React state usually avoids tearing because React controls the update flow.

- External stores need proper subscription handling to stay consistent with React rendering.

- Use `useSyncExternalStore` when connecting React to external stores safely.

- Libraries like Redux, Zustand, and React Query handle this internally when used correctly.

- Common mistake: reading external mutable values directly during render without subscribing properly.

- Interview trap: tearing is mostly about external state consistency, not normal prop or state updates.

- Real-world relevance: important for concurrent rendering, global stores, real-time data, and custom subscriptions.

## One-Line Final Recall

Tearing is an inconsistent UI state where components show different versions of external data during rendering.
