# Topic: Race Conditions

- Race conditions happen when multiple async operations finish in an unexpected order.

- Common React case: user triggers multiple API calls, but an older response updates the UI after a newer one.

- They often happen in `useEffect` when dependencies change quickly.

- Use cleanup logic in `useEffect` to ignore outdated async results.

- Use `AbortController` to cancel unnecessary fetch requests when possible.

- Always update state only if the response still belongs to the latest request.

- Race conditions can cause stale UI, wrong search results, flickering, or incorrect loading states.

- React Query, SWR, and RTK Query help manage many async race conditions automatically.

- Common mistake: assuming API responses will return in the same order they were called.

- Interview trap: race conditions are not React-specific; React just exposes them often through async UI updates.

## One-Line Final Recall

Race conditions happen when async results arrive out of order and update the UI with stale or incorrect data.
