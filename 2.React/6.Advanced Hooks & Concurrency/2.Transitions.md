# Topic: Transitions

- Transitions mark state updates as non-urgent so React can keep urgent updates responsive.

- Use `startTransition` when an update can be delayed, like filtering or rendering large UI.

- Urgent updates include typing, clicking, and direct input changes.

- Non-urgent updates include search results, tab content, charts, or expensive list rendering.

- `useTransition` gives `isPending` to show loading or pending UI during transition work.

- Transitions allow React to interrupt slower rendering if a more urgent update comes in.

- Do not use transitions for controlled input value updates; keep input state urgent.

- Common pattern: update input immediately, then transition expensive filtered results.

- Interview trap: transitions do not make code faster; they improve responsiveness by prioritizing work.

- Real-world use: search filtering, large lists, dashboards, tab switching, and slow UI updates.

## One-Line Final Recall

Transitions let React treat expensive UI updates as non-urgent so user interactions stay smooth and responsive.
