# Topic: Async Rendering

- Async rendering means React can prepare UI updates without blocking the main thread immediately.

- It helps React keep urgent interactions like typing and clicking responsive.

- React may pause, restart, or discard render work before committing the final UI.

- Async rendering mainly affects the render phase, not the commit phase.

- Components must be pure because render logic may run more than once.

- Avoid side effects during render; use `useEffect` or `useLayoutEffect` instead.

- Common async UI tools include `Suspense`, `useTransition`, and `useDeferredValue`.

- Async rendering does not mean JavaScript runs on multiple threads automatically.

- Common mistake: assuming async rendering makes slow components fast without optimization.

- Interview trap: async rendering improves scheduling and responsiveness, not raw execution speed.

## One-Line Final Recall

Async rendering lets React schedule and prepare UI work without blocking urgent user interactions.
