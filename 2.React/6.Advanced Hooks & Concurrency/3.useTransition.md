# Topic: useTransition

- `useTransition` lets you mark some state updates as non-urgent.

- It returns `[isPending, startTransition]`.

- Use `startTransition` for expensive UI updates like filtering, sorting, or rendering large lists.

- Urgent updates like typing and clicking should stay outside `startTransition`.

- `isPending` helps show loading or pending UI while the transition is running.

- Good pattern: update input value immediately, then transition the expensive result update.

- Transitions can be interrupted if a more urgent update happens.

- Common mistake: wrapping controlled input state updates inside `startTransition`.

- Interview trap: `useTransition` improves responsiveness, but it does not make expensive logic faster.

- Real-world use: search results, tab switching, dashboards, charts, and large list updates.

## One-Line Final Recall

`useTransition` keeps urgent UI updates responsive by marking expensive updates as non-urgent.
