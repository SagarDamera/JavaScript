# Topic: useDeferredValue

- `useDeferredValue` lets React delay updating a non-urgent value.

- It helps keep urgent UI updates responsive, like typing in an input.

- Use it when a value changes quickly but the expensive UI can update slightly later.

- Common use case: search input with large filtered results.

- The original value updates immediately, but the deferred value may lag behind.

- It works well when passing deferred values to expensive child components.

- `useDeferredValue` does not debounce API calls; it only defers rendering priority.

- Common mistake: using it instead of proper debouncing for network requests.

- Interview trap: `useDeferredValue` delays rendering work, while `useTransition` wraps state updates.

- Real-world use: search results, large lists, filtering, sorting, and heavy preview updates.

## One-Line Final Recall

`useDeferredValue` keeps the UI responsive by letting expensive updates use a slightly delayed value.
