# Topic: Concurrent Features

- Concurrent features help React keep the UI responsive during expensive or non-urgent updates.

- React can pause, resume, restart, or discard render work before committing changes.

- Main features include `useTransition`, `startTransition`, `useDeferredValue`, `Suspense`, and automatic batching.

- Urgent updates like typing and clicking are prioritized over slow UI updates.

- Non-urgent updates include filtering large lists, rendering charts, switching tabs, or loading heavy sections.

- Concurrent rendering affects the render phase, but the commit phase still completes fully once started.

- Components must stay pure because render logic may run more than once.

- Common mistake: thinking concurrent features make slow code faster; they mainly improve responsiveness.

- Interview trap: concurrent rendering is not parallel rendering with multiple browser threads.

- Real-world use: search pages, dashboards, large tables, lazy routes, and smoother async UI loading.

## One-Line Final Recall

Concurrent features let React prioritize urgent updates and prepare slower UI work without blocking user interactions.
