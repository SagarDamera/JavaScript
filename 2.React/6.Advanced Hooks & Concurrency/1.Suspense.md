# Topic: Suspense

- `Suspense` lets React show fallback UI while part of the component tree is waiting.

- Most common use is with `React.lazy()` for lazy-loaded components.

- Wrap lazy components inside `<Suspense fallback={...}>`.

- Fallback UI should be simple, like a spinner, skeleton, or loading text.

- Use multiple Suspense boundaries so different sections can load independently.

- Avoid wrapping the whole app in one large boundary if only one small area is loading.

- Suspense helps improve perceived performance, not actual API speed.

- Common mistake: forgetting fallback UI or making the fallback too heavy.

- Interview trap: Suspense does not fetch data by itself; it handles loading states for supported async patterns.

- Real-world use: lazy routes, dashboards, modals, charts, and streaming/server-rendered UI.

## One-Line Final Recall

`Suspense` shows fallback UI while lazy or async parts of the React tree are waiting to load.
