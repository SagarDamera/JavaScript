# Topic: Context API

- Context API is used to share data across components without passing props manually at every level.

- Common use cases: theme, auth user, language, feature flags, and app settings.

- Create context using `createContext()` and provide data using `Context.Provider`.

- Consume context using `useContext()` in functional components.

- Context is best for shared data that many components need, not every piece of state.

- Updating context value causes all consuming components to re-render.

- Avoid putting frequently changing state in one large context.

- Split contexts by responsibility, like `AuthContext`, `ThemeContext`, and `LocaleContext`.

- Common mistake: using Context as a replacement for all state management.

- Interview trap: Context solves prop drilling, but it does not automatically optimize performance.

## One-Line Final Recall

Context API shares common data across components without prop drilling, but should be used carefully to avoid unnecessary re-renders.
