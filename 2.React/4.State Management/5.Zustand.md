# Topic: Zustand

- Zustand is a lightweight state management library for React.

- It is simpler than Redux and needs less boilerplate.

- State is stored in a hook-based store created using `create()`.

- Components can select only the state they need from the store.

- Selecting small state slices helps reduce unnecessary re-renders.

- Zustand does not require Provider setup for basic usage.

- It works well for shared UI state, auth state, filters, cart state, and app settings.

- Keep server data in tools like React Query; use Zustand mainly for client/global state.

- Common mistake: putting all state into Zustand instead of keeping simple local state in components.

- Interview trap: Zustand is global state management, but it does not replace good state design.

## One-Line Final Recall

Zustand is a simple hook-based global state library useful for shared client-side state with minimal boilerplate.
