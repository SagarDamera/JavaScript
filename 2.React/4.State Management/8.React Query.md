# Topic: React Query

- React Query manages server state like API data, caching, loading, errors, and refetching.

- Use `useQuery` for fetching data and `useMutation` for create, update, or delete operations.

- It avoids writing repeated `useEffect`, loading state, error state, and manual cache logic.

- Query keys identify cached data; make them stable and include variables like IDs or filters.

- React Query can refetch data automatically on window focus, reconnect, or interval.

- Use `staleTime` to control how long data is considered fresh.

- Use `invalidateQueries` after mutations to refetch related cached data.

- Common mistake: using React Query for local UI state like modal open/close or form input.

- Interview trap: React Query is not a global client state library; it is mainly for async server state.

- Real-world use: dashboards, tables, search results, user profiles, CRUD pages, and cached API data.

## One-Line Final Recall

React Query simplifies server-state management by handling fetching, caching, refetching, and synchronization with APIs.
