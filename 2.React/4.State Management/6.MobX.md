# Topic: MobX

- MobX is a state management library based on observable state and automatic reactivity.

- It tracks which components use which observable values and re-renders only those parts when values change.

- Main concepts: observable state, actions, computed values, and reactions.

- `observable` holds reactive state, and `action` is used to update that state.

- `computed` is used for derived values that depend on observable state.

- Components usually use `observer` so they react to observable state changes.

- MobX often needs less boilerplate than Redux, but can feel less explicit.

- Keep business logic organized in stores instead of spreading state updates across components.

- Common mistake: mutating observable state outside actions in stricter MobX setups.

- Interview trap: MobX uses mutable reactive state, while Redux usually follows immutable update patterns.

## One-Line Final Recall

MobX manages state through observable data and automatically updates components that depend on that data.
