# Topic: Prop Drilling

- Prop drilling means passing props through multiple component levels just to reach a deeply nested child.

- It is not always bad; it is fine for small or simple component trees.

- Problem starts when intermediate components do not use the props but only pass them down.

- Prop drilling can make components harder to maintain and refactor.

- Use component composition to reduce unnecessary prop passing.

- Use Context when many components need the same shared data.

- Use state management libraries only when state becomes large, complex, or shared across many areas.

- Common mistake: using Context or Redux too early for simple parent-child data flow.

- Interview trap: prop drilling is a design smell, not a React error.

- Real-world use: solving shared auth, theme, user settings, or layout data across deeply nested components.

## One-Line Final Recall

Prop drilling is passing data through many layers, and it should be solved only when it hurts readability or maintainability.
