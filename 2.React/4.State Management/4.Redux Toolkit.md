# Topic: Redux Toolkit

- Redux Toolkit is the modern recommended way to write Redux with less boilerplate.

- It provides helpers like `configureStore`, `createSlice`, `createAsyncThunk`, and RTK Query.

- `createSlice` combines action creators and reducers in one place.

- Reducer logic can look mutable because Redux Toolkit uses Immer internally.

- `configureStore` sets up Redux store with good defaults like DevTools and middleware.

- Use `createAsyncThunk` for async logic like API calls when not using RTK Query.

- Use RTK Query for data fetching, caching, loading states, and API state management.

- Keep local UI state in components; use Redux Toolkit for shared or complex app state.

- Common mistake: putting every small state value into Redux instead of keeping it local.

- Interview trap: Redux Toolkit still follows Redux principles, but removes most manual boilerplate.

## One-Line Final Recall

Redux Toolkit is the standard way to use Redux efficiently with slices, simpler reducers, async helpers, and better defaults.
