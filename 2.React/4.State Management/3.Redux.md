# Topic: Redux

- Redux is a predictable global state management library for shared application state.

- Use Redux when many components need the same state or state updates are complex.

- Main parts: store, actions, reducers, dispatch, and selectors.

- State is updated by dispatching actions; reducers return the new state.

- Reducers must be pure and should never mutate existing state directly.

- Use Redux Toolkit in modern React; it reduces boilerplate and is the recommended approach.

- Use selectors to read specific parts of state and avoid unnecessary component coupling.

- Do not put all component state in Redux; keep local UI state inside components when possible.

- Common mistake: using Redux for simple state like modal open/close or input value.

- Interview trap: Redux is not required for every React app; Context or local state may be enough.

## One-Line Final Recall

Redux manages complex shared state predictably using a central store, actions, reducers, and selectors.
