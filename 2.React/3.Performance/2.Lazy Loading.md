# Topic: Lazy Loading

- Lazy loading means loading code only when it is needed, not during the initial app load.

- Use `React.lazy()` to load components dynamically.

- Wrap lazy-loaded components with `Suspense` to show fallback UI while loading.

- Lazy loading helps reduce initial bundle size and improve first-page load performance.

- Best use case: routes, dashboards, modals, heavy charts, and rarely used components.

- Avoid lazy loading very small or frequently used components because it may add unnecessary delay.

- `Suspense` fallback should be simple, like a spinner, skeleton, or loading text.

- Common mistake: forgetting `Suspense`, which causes lazy-loaded components to fail.

- Interview trap: lazy loading improves initial load, but the component still needs to download when first used.

- Real-world use: route-based code splitting in large React applications.

## One-Line Final Recall

Lazy loading improves performance by loading components only when they are actually needed.
