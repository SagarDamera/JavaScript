# Topic: Code Splitting

- Code splitting breaks the JavaScript bundle into smaller chunks loaded only when needed.

- It improves initial load performance by reducing the amount of code downloaded upfront.

- Common tools: dynamic `import()`, `React.lazy()`, and bundlers like Webpack or Vite.

- Best use case: route-level splitting, large pages, admin panels, charts, editors, and rarely used features.

- Use `Suspense` with lazy-loaded components to show fallback UI while the chunk loads.

- Avoid splitting too many tiny components; extra network requests can hurt performance.

- Code splitting does not remove code; it delays loading until the user needs it.

- Common mistake: lazy loading above-the-fold critical UI, causing visible loading delays.

- Interview trap: lazy loading is one technique; code splitting is the broader performance strategy.

- Real-world use: improving startup time in large React apps by loading feature code on demand.

## One-Line Final Recall

Code splitting improves React app performance by loading smaller JavaScript chunks only when needed.
