# Topic: Suspense

- `Suspense` lets React show fallback UI while something is loading.

- Common use: wrapping `React.lazy()` components for lazy loading.

- Use `fallback` to display a spinner, skeleton, or loading message.

- `Suspense` improves user experience by handling loading states declaratively.

- Place `Suspense` boundaries around parts of UI that can load independently.

- Avoid wrapping the whole app with one large `Suspense` boundary if only a small section is loading.

- Multiple `Suspense` boundaries allow different parts of the page to load separately.

- Common mistake: using lazy-loaded components without a `Suspense` fallback.

- Interview trap: `Suspense` does not fetch data by itself; it only handles waiting UI for supported async rendering patterns.

- Real-world use: lazy-loaded routes, modals, charts, dashboards, and async UI sections.

## One-Line Final Recall

`Suspense` shows fallback UI while lazy or async parts of the React tree are waiting to load.
