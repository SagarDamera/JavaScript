# Topic: Bundle Optimization

- Bundle optimization means reducing JavaScript size so the app loads faster.

- Use code splitting to load only the code needed for the current page or feature.

- Lazy load heavy components like charts, editors, dashboards, and rarely used modals.

- Remove unused dependencies and avoid importing large libraries for small utilities.

- Prefer tree-shakable imports instead of importing an entire library.

- Analyze bundle size using tools like bundle analyzer to find heavy packages.

- Avoid duplicate dependencies and unnecessary polyfills.

- Common mistake: optimizing React code but ignoring large third-party libraries.

- Interview trap: smaller bundle improves initial load, but runtime performance still depends on rendering efficiency.

- Real-world use: improving Core Web Vitals, startup time, mobile performance, and user experience.

## One-Line Final Recall

Bundle optimization reduces unnecessary JavaScript so React apps load faster and perform better.
