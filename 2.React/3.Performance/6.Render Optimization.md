# Topic: Render Optimization

- Render optimization means reducing unnecessary renders and expensive UI updates.

- Keep state as close as possible to where it is used.

- Avoid placing frequently changing state too high in the component tree.

- Use `React.memo` to prevent child re-renders when props do not change.

- Use `useMemo` for expensive calculated values, not for every small calculation.

- Use `useCallback` when passing functions to memoized child components.

- Avoid creating new objects, arrays, or functions inline when they cause memoized children to re-render.

- Use stable keys in lists to prevent incorrect component reuse and extra rendering work.

- Common mistake: optimizing before measuring; use React DevTools Profiler first.

- Interview trap: re-render is not always bad; unnecessary or expensive re-renders are the real problem.

## One-Line Final Recall

Render optimization focuses on preventing unnecessary or expensive renders while keeping React code simple and correct.
