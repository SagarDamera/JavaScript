# Topic: React.memo

- `React.memo` is used to memoize a component and skip unnecessary re-renders.

- It re-renders only when the component’s props change.

- Best use case: child components that receive the same props often and are expensive to render.

- `React.memo` does shallow comparison of props by default.

- Objects, arrays, and functions may still cause re-renders because their references change.

- Use `useMemo` and `useCallback` with `React.memo` when passing derived values or functions.

- Do not wrap every component with `React.memo`; it adds comparison cost and complexity.

- Common mistake: expecting `React.memo` to stop re-renders caused by local state or context changes.

- Interview trap: `React.memo` memoizes components, while `useMemo` memoizes values.

- Real-world use: optimizing large lists, table rows, cards, and expensive child components.

## One-Line Final Recall

`React.memo` skips unnecessary child re-renders by reusing the component output when props have not changed.
