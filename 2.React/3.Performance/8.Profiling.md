# Topic: Profiling

- Profiling means measuring React performance to find slow renders and unnecessary re-renders.

- Use React DevTools Profiler to see which components rendered and why.

- Always measure before optimizing; guessing often leads to useless complexity.

- Focus on components with high render time or frequent unnecessary renders.

- Check if props, state, context, or parent renders are causing re-renders.

- Use profiling results to decide where `React.memo`, `useMemo`, or `useCallback` is actually useful.

- Profile production-like builds because development mode can show slower or extra renders.

- Common mistake: optimizing small components that do not affect real user performance.

- Interview trap: profiling finds the problem; memoization is only one possible solution.

- Real-world use: improving slow pages, large tables, search filters, dashboards, and complex forms.

## One-Line Final Recall

Profiling helps identify real React performance issues before applying optimization techniques.
