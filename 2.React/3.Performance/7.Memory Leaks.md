# Topic: Memory Leaks

- Memory leaks happen when unused resources are not cleaned up after a component unmounts.

- Common causes: timers, subscriptions, event listeners, WebSocket connections, and pending async tasks.

- Always return a cleanup function from `useEffect` when creating external side effects.

- Clear timers using `clearTimeout` or `clearInterval`.

- Remove event listeners in cleanup to avoid duplicate handlers and stale references.

- Cancel or ignore async API results if the component unmounts before the response finishes.

- Be careful with subscriptions like RxJS, Firebase, sockets, or custom observers.

- Common mistake: starting side effects in `useEffect` but not cleaning them up.

- Interview trap: React re-render itself is not a memory leak; uncleaned external resources usually cause leaks.

- Real-world impact: memory leaks can slow the app, cause duplicate API calls, and create unexpected UI bugs.

## One-Line Final Recall

Memory leaks in React usually happen when effects create external resources but do not clean them up properly.
